# hello-ebpf Cookbook Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the single `docs/cookbook.md` with a layered, task-oriented documentation source at `docs/cookbook/` that covers the modern hello-ebpf feature set (verifier hints, maps, programs, kernel access, diagnostics), migrates the 16 existing topical docs into it, and preserves the file-path invariant that lets `VerifierFixSuggester` link `See: cookbook Â§X` to a real page anchor.

**Architecture:** 47 published `.md` pages plus a mirrored `_backing/` tree of 47 rationale/references files. Every page follows a layered template (TL;DR / Recipe / Full example / Why / Pitfalls / See also). Reference pages are symbol-oriented and generated-friendly; recipe pages are task-oriented and hand-written. Deep-dive pages replace the old topical guides one-for-one. A tiny JUnit sample module (`bpf-samples/â€¦/cookbook/`) compiles the recipe snippets against the real plugin, tagged so the docs build can require them green.

**Tech Stack:** MkDocs Material (existing), `pymdownx.superfences`/`snippets` for shared code fragments, JUnit 5 tags for recipe-vs-full-example separation, `mkdocs-macros` (optional, deferred) for `{{ recipe("name") }}` injection.

## 1. Design principles

1. **Recipe vs. reference vs. deep-dive** â€” three distinct page shapes, each has its own template. Never mix them.
2. **Layered structure** â€” every page opens with a one-paragraph TL;DR that a reader-in-a-hurry can copy; the deep material is below the fold.
3. **Verifier-hint file-path invariant** â€” every `See: cookbook Â§X` in `VerifierFixSuggester` maps to a stable page + anchor. Section renames require a corresponding suggester patch; the migration commit does both together.
4. **Backing docs** â€” every published page has a `_backing/<same-path>.md` with rationale, sources, blog-post links, and open questions. Backing docs are committed but excluded from `nav`.
5. **Existing docs migrate before new content is written** (Phase 0b). Migration is a rename + minor edit, not a rewrite. New material is layered on top of a stable nav.
6. **Compile every code block** â€” snippets under `bpf-samples/src/test/java/â€¦/cookbook/` are compiled as part of the normal Maven build; `@Tag("cookbook-recipe")` tags them so we can require green before publishing.
7. **Voice** â€” terse, active, second-person ("You writeâ€¦", "The verifier rejectsâ€¦"). No emojis. No marketing prose. Blog posts are the register to match (see `_backing/VOICE.md`).

## 6. Voice guidelines (`_backing/VOICE.md`)

Match the blog. Concrete markers derived from posts 1, 6, 9, 10, 14.5, 15, 16, 17:

- Second person, active voice. "You annotateâ€¦", "The verifier rejectsâ€¦".
- Terse but never terse-and-arch. Say what the thing does, then why.
- Kernel source references use file + line + version: `resolve_pseudo_ldimm64 in kernel/bpf/verifier.c (v6.17)`. Not just "the kernel does X".
- Verifier / clang / libbpf are named actors with observable behavior. "clang BPF strips the asm hint" beats "the compiler removes it".
- Every failure mode gets its exact error message quoted.
- No emojis. No "Let'sâ€¦", no "Soâ€¦", no exclamation points.
- Two audiences per page: a hurry (TL;DR + Recipe) and a builder (Why + Pitfalls). Serve both, in that order.

## 9. Testing conventions

- New sample tree: `bpf-samples/src/test/java/me/bechberger/ebpf/samples/cookbook/`.
- Each recipe class ends with `-Recipe.java`; each Full example ends with `-Example.java`.
- JUnit tag `cookbook-recipe` on recipe smoke tests; `cookbook-example` on full-example runs.
- MkDocs `pymdownx.snippets` imports code fences from the sample tree, so the doc is always in lockstep with the compiled version.
- Recipes that require a real kernel are gated on the same `@RequiresKernel(...)` pattern the existing samples use; docs build does NOT require a kernel â€” only compilation.
